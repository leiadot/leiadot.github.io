【 JS30 】Day08：Fun with HTML5 Canvas

2018-02-15
JavaScript
JS 30day JavaScript

javascript 30day

目標：拖拉滑鼠繪製網頁，且渲染顏色及動態變更筆劃粗細。

一、指定變數，並設定基本屬性

操作 html 的canvas標籤，先指定ctx變數為操作 canvas 的變數，
宣告ctx為2D渲染，並使canvas寬高和視窗寬高一致。

  • strokeStyle：線條顏色。
  • lineJoin：線條連接處。
  • lineWidth：線條寬度。

二、監聽事件

宣告isDrawing = false，判斷是否在畫圖的狀態，
且宣告lastX和lastY表示X軸、Y軸的起始位置。

在滑鼠移開或是放開按鍵的時候，使draw()判斷變數isDrawing為false，canvas則不具有作畫狀態；
在按壓狀態使isDrawing變數為true，draw()則判斷為作畫狀態；
滑鼠滑動時isDrawing變數為false，則不具有作畫狀態，但按壓且同時滑動，
因按壓時isDrawing變數為true，則不會中斷draw()，如此一來便可以達到滑鼠拖曳時具有作畫狀態的監聽。

三、開始作畫

  • beginPath：建立開始路徑
  • moveTo：將新路徑的起點移動到座標lastX、lastY。
  • lineTo：將起始點連結到目前的X、Y座標。（並未繪製）
  • stroke：描繪路徑。

使用解構賦值，將滑鼠當前位置賦予至作畫時X、Y軸的起始位置，
但分段作畫，上一段畫作的座標終點會和下一段畫作的座標起點連成直線，
為了避免這樣的狀況，所以必須修改監聽。

如此，便可分段每次按壓的狀況。

四、線段美化

宣告色相及方向的變數，並指定canvas顏色為hsl（色相/飽和/亮度）模式，
並在色相的部份嵌入模板變數，並使用遞增運算子，使色相有變化效果，
但因色相僅有 360 度，為了讓數值不要不斷累加上去，則大於等於 360 ，色相度數便歸零。

direction為true時，線條寬度將不斷遞增，
但大於等於 100 時，direction為false，便開始遞減，
遞減數值小於等於 1 時，direction為true，開始遞增，無限循環。
