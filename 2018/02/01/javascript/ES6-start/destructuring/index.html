陣列的解構賦值

基本用法

在 ES6 可以從陣列中提取值，按照對應位置，對變數賦值，
這種屬於模式匹配，兩邊等號的模式相同，左邊的變數就會被賦予對應的值。

使用set，也能使用陣列的解構賦值。

只要某種資料結構是屬於可以遍歷，都可以採用陣列形式的解構賦值。
fibs();是一個 Generator 函式，原生具有可遍歷的界面，解構賦值會依次從這個介面獲取值。

不完全解構

預設值

解構賦值允許指定預設值。

ES6 中使用===嚴格相等運算子來判定位置是否有值，
如果數組不嚴格等於undefined，預設值是無法生效。

如果一個數組成員是null，預設值就不會生效，因為null不嚴格等於undefined。

物件的解構賦值

物件的解構與陣列有一個重要的不同。陣列的元素是按次序排列的，變數的取值由它的位置決定；
而物件的屬性沒有次序，變數必須與屬性同名，才能取到正確的值。

物件的解構也能設定預設值。

預設值生效的條件是，物件的屬性值嚴格等於undefined。
如果x屬性等於null，就不嚴格相等於undefined，導致預設值不會生效。

如果解構失敗，變數的值等於undefined。

如果解構模式屬於巢狀物件，且子物件所在的父屬性不在，那將會回傳錯誤。
等號左邊物件的foo屬性，對應一個子物件。該子物件的bar屬性，解構時會報錯。
因為foo這時等於undefined，再取子屬性就會報錯。

第一個例子因為JavaScript引擎會將{x}理解成一個程式碼塊，而發生錯誤。

字串的解構賦值

字串被轉換為類似陣列的物件。

類似陣列的物件都有一個length屬性，因此還可以對這個屬性解構賦值。

數值和布林值的解構賦值

解構賦值時，等號右邊如果是布林或是數字，則會先轉成物件。
數值和布林值的包裝物件都有toString屬性，因此變數s都能取到值。

解構賦值的規則是，只要等號右邊的值不是物件或陣列，就先將其轉為物件。
由於undefined和null無法轉為物件，所以對它們進行解構賦值，都會報錯。

函式引數的解構賦值

函式的引數也可以使用解構賦值。
add()的引數表面上是一個數組，但在傳入引數的那一刻，陣列引數就被解構成變數x和y。
對於函式內部的程式碼來說，它們能感受到的引數就是x和y。

預設值

move();的引數是一個物件，對這個物件進行解構，得到變數x和y的值。
如果解構失敗，x和y等於預設值。

move();的引數指定預設值，而不是為變數x和y指定預設值，所以會得到與前一種寫法不同的結果。

undefined就會觸發函式引數的預設值。
